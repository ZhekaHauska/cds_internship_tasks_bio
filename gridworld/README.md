# Биологически правдоподобный EM-алгоритм для HMM

Одна из вычислительных [моделей (CSCG)](https://www.nature.com/articles/s41467-021-22559-5) формирования когнтитивных карт опирается на EM-алгоритм или алгоритм Баума-Велша для скрытой марковской модели (HMM). Данный алгоритм является очень общим и, как утверждается в статье, может быть реализован в мозге.

Предложите нейронную реализацию EM-алгоритма для HMM (как в CSCG), совместимую с основными известными [ограничениями](https://www.nature.com/articles/s41583-020-0277-3) в мозге. Вот некоторые из них:
1. Локальностью правил синаптической пластичности.
2. Асимметричностью входящих и исходящих связей.
3. Разделением нейронов на возбуждающие и ингибирующие.

Если соблюдение всех этих условий (1-3) невозможно, предложите упрощённый вариант EM алгоритма, который позволяет их соблюсти. Проанализируйте недостатки и преимущества такой биологически правдоподобной реализации.

**Эксперименты:**
1. Проверьте предсказательную способность полученной модели в частично наблюдаемой клеточной среде. В такой среде агенту неизвестно истинное состояние среды (координаты агента). Наблюдением, доступным агенту является цвет текущей текущей клетки и действие (проприоцепция). Позиция и цвет - категориальные дискретные случайные величины. Цвет клетки может дублироваться для разных позиций. То есть, число цветов меньше, чем число позиций (например, поле 10x10 c 10-ю цветами). **Используйте метрики:**
	1. Точность: совпадение наиболее вероятного наблюдения в предсказываемом распределении $I[argmax_j\ p(o_{t+1}=j\ |\ o_1, o_2, ..., o_{t}) = k]$, $I$ - индикаторная функция.
	2. Неожиданность: $-\log(p(o_{t+1}=k\ |\ o_1, o_2, ..., o_{t}))$, где $k$ - это наблюдаемый цвет.
2. Проанализируйте пространство скрытых состояний HMM. Можно ли скрытые состояния однозначно отнести к той или иной позиции в вашей модели и оригинальной CSCG? Как это связано с точностью предсказаний наблюдений и неожиданностью наблюдений?
3. Может ли ваша предложенная модель выучить структуру пространства среды? Постройте граф выученной структуры, как в статье про CSCG.

## Установка и запуск

### 1. Настройка окружения с помощью pyenv

```bash
# Установите pyenv, если его нет
curl https://pyenv.run | bash

# Установите Python 3.11
pyenv install 3.11

# Создайте виртуальное окружение в проекте
cd gridworld
pyenv local 3.11
python -m venv .venv

# Активируйте окружение
source .venv/bin/activate

# Установите зависимости
pip install -r requirements.txt
```

### 2. Запуск эксперимента

```bash
# Активируйте окружение (если ещё не активировано)
source .venv/bin/activate

# Запуск с конфигом по умолчанию
python start_script.py

# Или с указанием конфига через переменную окружения
RUN_CONF=configs/runner/gridworld_random.yaml python start_script.py
```

## Структура проекта

### configs/runner/gridworld_random.yaml
Основной конфиг запуска, связывающий все компоненты. Содержит:
- выбор логгера ([aim](https://aimstack.readthedocs.io/en/latest/quick_start/setup.html), null)
- пути к конфигам агента, окружения, сценария и метрик
- остальное можно оставить как есть

### configs/environment/gridworld/pomdp.yaml
Конфигурация частично наблюдаемой марковской модели (POMDP):
- `n_random_colors` — количество случайных цветов 
- остальное можно оставить как есть

### configs/environment/gridworld/setups/free_10x10_no_landmarks.yaml
Описание поля 10x10:
- `room[0]` — цвета клеток
- `room[1]` — награды на клетках
- `room[2]` — терминальные состояния
- `room[3]` — ориентиры (landmarks)
- `start_position` — начальная позиция агента (если null, то позиция выбирается случайно каждый эпизод)

### configs/agent/random/default.yaml
Конфигурация агента. Для `RandomAgent` пуст, но для вашего агента будет содержать его гиперпараметры.

### configs/scenario/gridworld_base.yaml
Сценарий выполнения — условия завершения эпизодов и логирования:
- `episodes == 1000` — остановить после 1000 эпизодов
- `steps == 100` — максимум 100 шагов в эпизоде
- `episodes == 0` — начать логирование с первого эпизода

Можно также добавлять вызов кастомных функций с root в объекте runner.

### configs/metrics/gridworld_pomdp.yaml
Конфигурация метрик:
- `agent/surprise` — неожиданность (метрика неожиданности)
- `agent/accuracy` — точность предсказаний
- Частота обновления и логирования

Сюда можно доавлять свои метрики.

Текущий `RandomAgent` (`my_lib/agents.py`) вычисляет предсказания через простую частотную статистику переходов по наблюдениям (`first_order_transitions`) (используйте как бейзлайн). Вам нужно заменить его на **собственного агента**, который делает предсказания `prediction` на основе вашей HMM-модели.

**Что нужно сделать:**

1. **Создайте новый класс** в `my_lib/agents.py`, унаследовав его от `BaseAgent` (`my_lib/base.py`).
2. **Реализуйте метод `observe()`** — в нём вычисляется `prediction` (распределение вероятностей следующего наблюдения). Метрики `accuracy` и `surprise` вычисляются так же, как в `RandomAgent`.
3. **Зарегистрируйте агента** в `TestRunner.make_agent()` (`start_script.py:11-17`), добавив новый `agent_type` (например, `'my_agent'`).
4. **Создайте конфиг** агента в `configs/agent/ваше_имя/default.yaml`.
5. **Обновите `configs/runner/gridworld_random.yaml`**: измените `agent_conf` на путь к вашему конфигу и `agent_type` на название вашего агента.

Ключевые атрибуты агента для реализации EM/HMM-алгоритма:
- `self.n_obs_states` — число наблюдаемых состояний (цветов)
- `self.n_actions` — число действий
- `self.obs_state` — текущее наблюдение (вызывается в `observe()`)
- `self.first_order_transitions` — можно использовать как базу для матрицы переходов HMM

**Базовый класс для редактирования:** `BaseAgent` в `my_lib/base.py` (интерфейс), `RandomAgent` в `my_lib/agents.py` (шаблон для реализации).
